# Week 8 Journal — Functional User Requirements

To create a functional website, developers must have a better understanding of what the user requirements are. That is when the user story comes in. The user story usually starts with "As a user" and explains what the user is expecting from the software under development. Knowing the user's needs and motivations will help the developer generate clear specifications for software features and uses. *(Week 8 Lecture Notes, 2026)*

---

## Guidelines to Write a Perfect User Story

1. **Keep it short:** The story should be short enough to fit on a sticky note; otherwise, it is not fit for purpose.
2. **Clear "Who":** Have a clear understanding of who the potential end user is.
3. **Clear "What":** The story should be clear to the client, end user, and developer so that everyone understands each role and why they want it.
4. **Not too big:** Ask yourself whether the feature described in the story can be broken into smaller chunks. If yes, consider whether it should be one story or many stories. To scope the story correctly, make sure it can be completed in ideally one week, but no more than two weeks. If it takes more than two weeks, it is too big and should be broken into smaller, more manageable chunks.
5. **Clear "Why":** Look for the root cause of the problem through root cause analysis. This could be the "Why" — the reason we are building this story. If there is no clear why, ask the client to articulate the root cause clearly.

*(Angela Kim, 2022)*

A user story describes a feature from the user's perspective, explaining what the user wants and why. A functional requirement describes the specific technical tasks or system functions needed to implement that user story. *(Angela Kim, 2022)*

## Choose a Programming Language to Research

For this assignment, I have chosen to research **Processing**.

Processing is a free, open-source programming software first released in **2001**. It was initially created to help new programmers learn coding through visual context. Over time, Processing evolved from a beginner-friendly tool into a fully capable development environment used by professionals.

Processing runs on Mac, Windows, and Linux, and is widely used in schools, universities, and programs focused on art, design, and computer science. Its open-source community actively contributes libraries covering areas such as data visualization, music, computer vision, and electronics, including integration with Arduino.

Artists, designers, and museums use Processing for interactive installations, visual art, live performances, and media projects, helping new generations integrate programming into their creative work. *(Reas & Fry, 2022)*

## Task 1 — User Story

> **Estimated time:** 20 minutes

For this task, I selected an app I use regularly and chose a feature to write user stories about.

**App:** Spotify  
**Feature:** Playlist creation and management

### General User Story

- As a user, I can create a playlist to organize and save songs I enjoy.

### More Specific User Stories

- As a user, I can create a new playlist from the home screen and give it a custom name.
- As a user, I can add a song to an existing playlist directly from the song's options menu.
- As a user, I can reorder songs within a playlist by dragging them into a preferred position.

### Acceptance Criteria — "Adding a song to an existing playlist"

- [ ] The song options menu includes an "Add to playlist" button.
- [ ] The playlist selection screen clearly displays all existing playlists.
- [ ] The most recently used playlist appears at the top of the list for quick access.
- [ ] A confirmation message briefly appears after the song has been successfully added.
